# 에이전트 응답속도는 모델 문제가 아니라 경로 문제다

AI Agent가 느리면 가장 먼저 떠오르는 해결책은 모델을 바꾸는 것이다. 더 빠른 모델, 더 작은 모델, 더 좋은 provider를 찾는다.

물론 모델 속도는 중요하다. 하지만 실제 제품에서 체감 latency를 크게 만드는 것은 모델 하나가 아닌 경우가 많다. 검색을 몇 번 도는지, 도구를 어떻게 고르는지, prompt가 얼마나 비대한지, cache가 먹히는지, 실패했을 때 같은 경로를 다시 도는지가 더 큰 차이를 만든다.

그래서 에이전트 응답속도는 “모델 속도”가 아니라 **실행 경로의 길이**로 봐야 한다.

## 먼저 latency budget을 나눈다

응답 하나가 40초 걸린다고 해서 40초를 전부 모델 탓으로 보면 안 된다. 보통은 여러 구간으로 나뉜다.

- 사용자 입력 해석
- retrieval 또는 DB lookup
- tool routing
- tool execution
- 중간 결과 압축
- 최종 generation
- 후처리와 formatting

이 구간을 나누지 않으면 최적화는 감으로 흐른다. 모델을 바꿨는데 retrieval이 그대로 느리면 사용자는 여전히 느리다고 느낀다. prompt를 줄였는데 tool loop가 세 번 돈다면 개선 폭은 작다.

## 자주 묻는 질문에는 fast path가 필요하다

모든 질문을 full reasoning path로 보내면 시스템은 똑똑해 보이지만 느려진다.

반복되는 질문, 데이터 lookup만으로 답할 수 있는 질문, 답변 형식이 안정적인 질문은 빠른 경로를 가져야 한다. 이 경로에서는 agent가 모든 도구를 탐색하지 않는다. 필요한 데이터 함수로 바로 가고, LLM은 설명을 다듬는 역할만 한다.

에이전트가 항상 깊게 생각해야 좋은 것은 아니다. 깊게 생각할 질문과 빠르게 처리할 질문을 나누는 것이 운영 품질이다.

## Tool loop는 latency를 기하급수적으로 키운다

에이전트가 느려지는 흔한 이유는 도구 호출 자체가 아니라 도구 호출의 반복이다.

- 비슷한 검색을 여러 번 한다.
- 실패한 tool path를 다시 시도한다.
- 이미 얻은 정보를 prompt에 넣지 않아 같은 확인을 반복한다.
- router가 불명확해서 매번 후보 도구를 탐색한다.

이 문제는 모델보다 contract 문제에 가깝다. 도구는 “무엇을 입력받고, 무엇을 보장하며, 언제 쓰지 말아야 하는지”가 명확해야 한다. route contract가 흐리면 agent는 느려지고, 로그는 길어지고, 사용자는 결과를 기다리다 이탈한다.

## Prompt slimming은 단순 절약이 아니다

Prompt를 줄이는 일은 토큰 비용 절감 이상의 의미가 있다. prompt가 짧아지면 모델 입력 시간이 줄고, 불필요한 지시 충돌이 줄고, agent가 지금 해야 할 일을 더 잘 본다.

특히 운영 중인 agent는 시간이 지나며 prompt가 비대해지기 쉽다. 예외 규칙, 임시 패치, 오래된 정책, 더 이상 쓰지 않는 도구 설명이 계속 붙는다. 이 상태에서 latency와 품질이 동시에 나빠진다.

Prompt slimming은 문장 삭제가 아니라 실행 경로 청소다.

## Cache는 똑똑함을 대체하지 않지만 반복을 줄인다

agent가 매번 새로 생각해야 할 필요는 없다. 자주 쓰는 시스템 지식, 변하지 않는 데이터 설명, 반복되는 변환 결과는 cache나 precomputed artifact로 빼는 편이 낫다.

좋은 cache는 모델의 지능을 대체하지 않는다. 대신 모델이 매번 같은 준비 운동을 하지 않게 만든다.

## 결론: 빠른 agent는 짧은 경로를 가진다

AI Agent latency를 줄일 때 첫 질문은 “어떤 모델이 더 빠른가?”가 아니다.

먼저 물어야 할 것은 이것이다.

- 이 질문은 full agent loop가 필요한가?
- retrieval과 tool execution 중 어디가 느린가?
- 같은 도구를 반복 호출하고 있지 않은가?
- prompt에 오래된 지시가 쌓이지 않았는가?
- cache로 뺄 수 있는 준비 작업은 없는가?

빠른 agent는 단순히 빠른 모델을 쓰는 agent가 아니다. **짧고 명확한 경로를 가진 agent**다.
